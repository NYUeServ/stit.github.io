DESIGN STANDARDS

A
When a brand new idea comes up. In other words, when we have to develop a new service/application/feature from scratch.
  • Define project scope
  • User research
  • Analyze & define solution
  • Create user flow
  • Wireframing
  • UI Design
  • Usability testing
  • Share with developers
  • Iteration
B
Solving existing issues from a UX/UI aspect (User flow/features optimization).
  • Research existing problem
  • Define project scope
    • A. Develop new features
    • A-1. Analysis & define solutions
    • A-2. Create user flow
    • A-3. Wireframing
    • A-4. UI Design
    • A-5. Usability testing
    • A-6. Share with developers
    • B. Optimize current user flow/feature
    • B-1. Usability testing
    • B-2. Analysis & define solutions
    • B-3. Create user flow
    • B-4. Wireframing
    • B-5. UI Design
    • B-6. Usability testing
    • B-7. Share with developers
C
Fast track. When the needs is very obvious and straightforward, directly start to design.
  • Confirm the needs
  • UI Design
  • Usability testing
  • Share with developers
